# Future Roadmap (今後の展望)

`esa-summarizer` のさらなる発展に向けたアイデア案です。

## 1. AI 機能の強化 (AI Capabilities)

*   **インタラクティブな Q&A 機能**
    *   要約に対してスレッドで質問すると、Bot が記事の内容に基づいて回答してくれる機能。
    *   例: 「この記事の結論は何？」「XXについて詳しくは書いてある？」
*   **要約スタイルのカスタマイズ**
    *   ユーザーがコマンドで要約のターゲットを指定できるようにする。
    *   例: `--target engineer` (技術的な詳細中心), `--target executive` (ビジネスインパクト中心)
*   **マルチモーダル対応**
    *   esa 記事内の画像や図表も Gemini に認識させ、よりリッチな要約を生成する。

## 2. 連携・ワークフローの深化 (Integration)

*   **双方向同期 (Two-way Sync)**
    *   Slack で生成された要約を、esa 記事のコメント欄にも自動で書き戻す。
    *   記事を見た人が、読む前に要約を確認できるようになる。
*   **ダイジェスト配信**
    *   毎日または毎週、その期間に更新された記事の「まとめ読みダイジェスト」を配信する。
*   **スマートルーティング**
    *   esa のタグやカテゴリ（例: `日報`, `仕様書`, `ポエム`）に応じて、投稿先の Slack チャンネルを自動で振り分ける。

## 3. 運用・信頼性向上 (Operations)

*   **コスト最適化 (Caching)**
    *   一度要約した記事の結果をキャッシュ（保存）し、同じ記事に対するリクエストで再度 AI を呼ばないようにしてコストを削減する。
*   **自動再試行 (Retry Logic)**
    *   esa や Gemini の API が一時的にエラーになった場合、自動的にリトライする仕組みを強化する。
*   **利用分析 (Analytics)**
    *   どの記事の要約がよく読まれているか（リアクション数など）を計測し、人気のナレッジを可視化する。

## 4. ユーザー体験 (UX)

*   **Slack Home Tab 対応**
    *   Bot のホームタブに、最近の要約一覧や、個人の設定画面を表示する。
*   **Slash Command 導入**
    *   `/esa-settings` のようなコマンドで、監視チャンネルや通知先を Slack 上から簡単に変更できるようにする。
